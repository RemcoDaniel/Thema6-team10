593  //
595  //
745  //
746  // A task is created by inheriting
747  // from task and providing a main() function. Initialisation of the task,
748  // including creating its waitables, should be done in the constructor.
749  // Don't forget to call the constructor of the task class!
750  //
751  // The main() is the body of the task. It should never terminate.
752  //
753  // Each task has a unique priority (an unsigned integer).
754  // A lower value indicates a higher priority.
755  // The RTOS scheduler will always run the task with the higest-priority
756  // runnable (neither blocked nor suspended) task.
757  // A task runs until it changes this 'situation' by using an RTOS
758  // call that changes its own state to not runnable, or the state of a
759  // higher priority task to runnable.
760  //
761  // Timers are served only when the RTOS is activated
762  // by calling any of its state-changing interfaces.
763  // Hence the longest run time between such calls determines the granularity
764  // (time wise responsiveness) of the application.
765  // Within a time consuming computation a task can call release() to have
766  // the RTOS serve the timers.
767  //
